5.0 Known Issues
  1. We do not intend to support PHP Version 3.

  2. The RETS server (or Source) must be up and operational for the package to operate. Some configuration can be performed off-line but as auto-detection capabilities are improved, live connections will become more important.

  3. STANDARD_XML response format from RETS Servers is not supported.

  4. Data cannot be sorted. This is a limitiation in the RETS specification, not a VieleRETS constraint
